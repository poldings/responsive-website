Responsive Website Template

A logically structured template that makes it easy to build responsive websites.

A responsive website is a website that responds according to the type or size of browser being used to view it.

There are so many different browser types available that it can be an onerous task for any web developer to make a website work across all of them ...

  • smartphones
  • phablets
  • tablets
  • laptops
  • desktops

To succeed online, a business website needs to look great and work perfectly well across all technology mediums - a truly responsive website will do just that.

 

The ideal website

As a web developer, I like clean, lean websites that have not been bogged down by lots of unnecessary code.

I love HTML-5, CSS-3 and jQuery but they can add a lot of extra code and need extra work to make them compatible with all the browser types.

For me, a website should be functional not flamboyant. It should be practical and straightforward and do what it needs to in the shortest number of clicks.

"Don't do it just because you can. Do it because it adds value."
Vincent Polding, Web Developer at Poldings Ltd
 
 

The fluid template

This template isn't fancy and it's not going to take your breath away - it's just a very simple way to build a website that will adapt across all browser types and sizes.

Unlike a lot of responsive frameworks, it uses percentages instead of a grid system - making it easy to understand and reproduce.

Consider the width of each division (div) to be a maximum of 100% of its parent division - you can begin to divide that division into sections using width percentages.

Code Snippet One shows how to produce two equal columns of 45% width with a 10% space between them.

<div class='col45'> Content of first column. </div>

<div class='col10'> &nbsp; </div>

<div class='col45'> Content of second column. </div>

<div class='clear'> &nbsp; </div>
Code Snippet One
 

Making things dis/appear

Due to it's size and functionality, a website should look different on a mobile device and for this purpose, you will need to change elements of your page content and layout.

To do this, we use CSS to make things appear and disappear as the browser size changes.

Code Snippet Two shows how to show and hide content according to whether a smartphone browser is being used.

<div class='hideif'> Not using a smartphone. </div>

<div class='showif'> Smartphone being used. </div>
Code Snippet Two
 

Optimised for smartphones

The CSS for this project has been set to recognise when a browser reaches a width of 480px or less (smartphone browser territory).

At this point it takes all columns and, regardless of their size, forces them to 100% width of the parent div.

So, in the Code Snippet One example above, the column styled to a width of 45% ('col45') will now be styled to a width of 100%.

Although you can modify the CSS to react to other browser sizes, I find that for most projects, it's only necessary to adjust the layout when it reaches 480px.

 

One size fits all

Don't you hate it when images are either too large or too small for their purpose? With responsive websites, getting images the right size all the time can be a bit of an issue.

Code Snippet Three shows how to add a little CSS styling to an image so that it resizes as the page expands and contracts.

<img src='/evolution.png' alt='Evolution' class='maxwidth' />
